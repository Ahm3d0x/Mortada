===============================================================
شرح منطق الكمبيوتر (الذكاء الاصطناعي) بالتفصيل الكامل
===============================================================
الملف المرجعي: src/components/GameOffline.tsx
الدوال الرئيسية: handleAIPlayTurn, triggerAIDefenseReaction, handleConfirmDefense

---------------------------------------------------------------
قسم 1: نظرة عامة على الكمبيوتر
---------------------------------------------------------------

الكمبيوتر في اللعبة له هوية ثابتة:
  aiCoachName = "المدرب الغريم (تكتيك روبوت)"
  aiTeam      = "كتائب الروبوت الذكية 🤖"

الكمبيوتر ليس ذكاءً اصطناعياً حقيقياً يتعلم أو يُفكر،
بل هو محرك قواعد حتمي يتبع خوارزمية ثابتة في كل دور.

يعمل على مرحلتين رئيسيتين:
  أ) ai_turn         → الكمبيوتر يلعب دوره الهجومي كاملاً تلقائياً
  ب) دفاع تلقائي    → الكمبيوتر يدافع تلقائياً عند تسديد اللاعب

---------------------------------------------------------------
قسم 2: دور الكمبيوتر الهجومي (handleAIPlayTurn)
---------------------------------------------------------------

تُطلق هذه الدالة تلقائياً بمجرد أن تصبح المرحلة ai_turn.
لا يلعب الكمبيوتر في وضع الأونلاين (isMultiplayer) ولا في استراحة الشوط.

التسلسل الزمني:
  التشغيل الفوري → تأخير 1200ms → سحب الكروت → تأخير 1200ms آخر → الحركات الكاملة

== الخطوة 1: سحب الكروت ==
الكمبيوتر يسحب ورقتين دائماً:
  - ورقة لاعب من aiDeck (أول ورقة في المجموعة)
  - ورقة تكتيك خاصة من specialDeck (أول ورقة)
  ملاحظة مهمة: السحب دائماً من أول الكارت، لا عشوائية هنا.

== الخطوة 2: تنظيم التشكيلة (Phase 1: Layout Support) ==

الكمبيوتر يُحسّن مراكز الملعب بالأولوية:

أ) مركز فارغ موجود؟
   → يُرتب كروت اليد (لاعبين) حسب مجموع (هجوم + دفاع) تنازلياً
   → يضع الأفضل في المركز الفارغ مقلوباً

ب) لا يوجد فارغ لكن يوجد مركز مستهلك (spent)?
   → يضع الأفضل مكان المستهلك ويُعيد اللاعب القديم للمجموعة (recycle)

ج) لا يوجد فارغ ولا مستهلك؟
   → يبحث عن أضعف لاعب: (attack < 6) و (defense < 6)
   → يتحقق: هل الجديد أفضل بنقطتين (+2) على الأقل؟
   → إذا نعم: يستبدل
      * إذا كان الضعيف مكشوفاً → يُطرد نهائياً من الملعب
      * إذا كان مقلوباً → يعود ليد الكمبيوتر
   → إذا لا: يوقف البحث (break)

== الخطوة 3: تفعيل الكروت التكتيكية ==

لكل كارت تكتيكي (special) في اليد:

أ) كروت الاستهداف (Destroy/Freeze/Silence/Stun/ReturnToHand/Reveal):
   → يبحث عن أقوى لاعب مكشوف للاعب الإنسان
   → يُرتب حسب (هجوم + دفاع) تنازلياً
   → يطبق التأثير على الأقوى
   → يُزيل الكارت من اليد ويستهلك حركة

ب) كروت السحب (DrawCard / world_cup):
   → يسحب عدداً من الكروت الإضافية
   → يستهلك حركة واحدة

ج) قدرات الأسطورة عند الوضع في الملعب:
   → AddMoves: يزيد عدد حركات الكمبيوتر في هذا الدور
   → DrawCard: يسحب كروتاً إضافية فوراً

== الخطوة 4: إعلان الهجوم (Phase 2: Attack Declaration) ==

بعد تنظيم التشكيلة، يبحث الكمبيوتر عن مهاجم:

المرشحون للهجوم = اللاعبون المقلوبون + غير المستهلكين + هجومهم > 1
الترتيب = تنازلياً حسب قوة الهجوم (الأعلى هجوماً أولاً)

شروط الهجوم:
  - يوجد مرشح واحد على الأقل
  - يوجد حركة واحدة على الأقل متبقية
  - الهجوم غير محظور (BlockAttack)

عند تنفيذ الهجوم:
  1. يكشف اللاعب المهاجم (يقلبه وجهاً للأعلى)
  2. يسحب كارت Booster عشوائي (معزز المرتدة)
  3. ينقل المرحلة إلى ai_attacking
  4. يُتاح للاعب الإنسان حركات الدفاع (maxMovesPerTurn)
  5. يُحرر قفل التنفيذ ليتمكن اللاعب من الدفاع

إذا لا يوجد هجوم متاح أو نفدت الحركات → ينهي الدور (handleEndAITurn)

---------------------------------------------------------------
قسم 3: دفاع الكمبيوتر ضد هجوم اللاعب (triggerAIDefenseReaction)
---------------------------------------------------------------

يُستدعى تلقائياً عند ضغط اللاعب على "تسديد/حسم الهجوم"
بعد تأخير 1200ms.

== الخطوة 1: حساب فجوة الدفاع ==

هجوم اللاعب الكلي = قيمة المهاجم + Booster + كروت تكتيكية + داعمون + قدرات
دفاع الكمبيوتر الحالي = مجموع دفاع اللاعبين المكشوفين + كروت تكتيكية

فجوة الدفاع = هجوم اللاعب - دفاع الكمبيوتر الحالي

إذا فجوة > 0 → الكمبيوتر يحتاج للدفاع أكثر
إذا فجوة ≤ 0 → الكمبيوتر مطمئن، لا يكشف المزيد

== الخطوة 2: تقييم الكروت التكتيكية الدفاعية ==

إذا كانت فجوة الدفاع موجودة، يُرتب الكمبيوتر الكروت التكتيكية بالأولوية:

1. Offside / CancelAction  → يلغي الهجوم كلياً (أعلى أولوية!)
2. FreezeCard/SilenceCard/StunCard/DestroyCard → يُضعف/يُحذف المهاجم
3. park_the_bus / AddStat(defense ≥ 4) → رفع دفاع كبير
4. wet_pitch / AddStat(attack ≤ -4) → تخفيض هجوم اللاعب
5. fans / AddStat(≥ 2) → تعزيز بسيط

عند اختيار الكارت:
  → يطبقه فوراً على المهاجم الحالي للاعب (إذا كان استهداف)
  → يُعيد حساب الدفاع بعد كل كارت
  → يستمر حتى: فجوة الدفاع ≤ 0 أو نفدت الحركات أو نفدت الكروت

== الخطوة 3: خوارزمية اختيار أفضل مزيج دفاعي ==

بعد الكروت التكتيكية، يحسب الكمبيوتر الفجوة المتبقية
ويحاول إيجاد أفضل مزيج من اللاعبين المقلوبين بالترتيب:

محاولة 1: لاعب واحد
  → يبحث عن أقل لاعب دفاعه >= فجوة الهجوم
  → يختار الأقل كفاية (لا يُبدد قوة زائدة)
  → إذا وجد: ينتهي البحث ✓

محاولة 2: زوج من اللاعبين
  → يجرب كل الأزواج الممكنة
  → يختار الزوج الذي (مجموع دفاعهم >= فجوة) بأقل فائض
  → إذا وجد: ينتهي البحث ✓

محاولة 3: ثلاثة لاعبين
  → يجرب كل التوليفات الثلاثية
  → نفس منطق الزوج

محاولة 4: أربعة لاعبين
  → يجرب كل التوليفات الرباعية
  → نفس المنطق

Fallback (إذا فشل الكل):
  → يُرتب المرشحون حسب الدفاع تنازلياً
  → يكشف أعلى عدد مسموح به منهم
  → حتى لو لم يُوقف الهجوم، يحاول الحد من الأضرار

لا يوجد مدافعون متاحون:
  → يطبع: "الخصم أدرك استحالة الصد وفضّل الحفاظ على أوراقه!"
  → لا يكشف أي لاعب

قيد مهم:
  عدد الكشوفات المسموحة = maxMovesPerTurn - عدد الكروت التكتيكية المُفعَّلة
  إذا كان الدفاع محظوراً (BlockDefense) → عدد الكشوفات = 0

---------------------------------------------------------------
قسم 4: لحظة الحسم (handleConfirmDefense)
---------------------------------------------------------------

عندما يضغط اللاعب على "تأكيد الدفاع":

== تعزيز هجومي للكمبيوتر (قبل النتيجة) ==
إذا كان هجوم الكمبيوتر <= دفاع اللاعب (أي الكمبيوتر سيخسر):
  → يُضيف الكمبيوتر كروت تكتيكية هجومية من يده واحدة بواحدة
  → يُعيد حساب الهجوم والدفاع بعد كل كارت
  → يستمر حتى: الهجوم > الدفاع، أو نفدت الحركات، أو نفدت الكروت

== معادلة الحسم النهائية ==

هجوم الكمبيوتر = قيمة المهاجم + Booster + تأثيرات تكتيكية + لاعبون داعمون + قدرات خاصة
دفاع اللاعب   = مجموع دفاع المدافعين المكشوفين + تأثيرات تكتيكية + قدرات خاصة

النتيجة:
  هجوم الكمبيوتر > دفاع اللاعب → هدف للكمبيوتر 🥅
  هجوم الكمبيوتر ≤ دفاع اللاعب → إنقاذ للاعب 🧤

== تعزيز الهجوم المضاعف (AI Reinforce Attack) ==

إذا أُحبط هجوم الكمبيوتر لكن لا يزال لديه حركات:

الشروط:
  - لا يعمل في وضع أونلاين
  - حركات الكمبيوتر المتبقية > 0
  - إجمالي الأفعال في هذه الجولة < maxMovesPerTurn
  - يوجد لاعبون مقلوبون غير مستهلكين

الإجراء:
  → يختار أعلى لاعب هجوماً من المقلوبين غير المستهلكين
  → يكشفه ويضيفه للهجوم
  → يقفل مدافعي اللاعب الحاليين (لا يمكن سحبهم)
  → ترجع المرحلة إلى ai_attacking
  → اللاعب يستطيع إضافة مدافعين جدد وتأكيد الدفاع مجدداً

---------------------------------------------------------------
قسم 5: محرك حساب القوة (rulesEngine.ts)
---------------------------------------------------------------

يحسب runRefereeRulesEngine قوة الهجوم أو الدفاع الفعلية.

مكونات قوة الهجوم:
  + card.attack (قيمة اللاعب الأساسية)
  + currentBooster.value (قيمة المعزز العشوائي)
  + AddStat(attack) من الكروت التكتيكية
  + مجموع attack للاعبين الداعمين المكشوفين
  + تأثيرات قدرات الأسطورة

مكونات قوة الدفاع:
  + مجموع card.defense للمدافعين المكشوفين
  + AddStat(defense) من الكروت التكتيكية
  + park_the_bus / BlockAttack
  + تأثيرات قدرات الأسطورة

---------------------------------------------------------------
قسم 6: القيود والضمانات التقنية
---------------------------------------------------------------

isAIExecutingRef   → يمنع تشغيل دورين للكمبيوتر في نفس الوقت
isHalfTimeBreakRef → يوقف الكمبيوتر فوراً عند استراحة الشوط
aiTimeoutId1/2     → مراجع للـ timeouts يمكن إلغاؤها عند الحاجة
maxMovesPerTurn    → سقف الحركات في كل دور (يشمل كشوفات + تكتيكية)
isAttackBlockedFor → يتحقق إذا كان الهجوم محظوراً بكارت تكتيكي
isSpecialCardsBlocked → يتحقق إذا كانت الكروت التكتيكية محظورة

---------------------------------------------------------------
قسم 7: الخوارزمية الكاملة (ملخص مرئي)
---------------------------------------------------------------

بداية دور الكمبيوتر (ai_turn)
        │
        ▼ [1200ms]
◆ سحب ورقتين (لاعب + تكتيك)
        │
        ▼ [1200ms]
◆ تنظيم التشكيلة:
  └─ فارغ → أفضل لاعب
  └─ مستهلك → استبدال + إعادة تدوير
  └─ ضعيف → استبدال إذا الجديد أفضل +2
        │
◆ تفعيل الكروت التكتيكية:
  └─ استهداف → أقوى لاعب مكشوف للاعب
  └─ سحب → كروت إضافية
        │
◆ إعلان الهجوم:
  └─ يوجد مقلوب بهجوم>1 → اختيار الأعلى هجوماً
  └─ كشفه + Booster → ai_attacking
  └─ لا يوجد → إنهاء الدور
        │
   [عند تسديد اللاعب]
        │
        ▼ [1200ms]
◆ دفاع الكمبيوتر:
  └─ حساب فجوة الدفاع
  └─ كروت تكتيكية بالأولوية
  └─ بحث عن مزيج 1→2→3→4 لاعبين
  └─ fallback: الأقوى دفاعاً
        │
   [عند تأكيد الدفاع من اللاعب]
        │
◆ لحظة الحسم:
  └─ كمبيوتر يُضيف كروت هجومية (إذا كان سيخسر)
  └─ هجوم > دفاع → هدف 🥅
  └─ هجوم ≤ دفاع → إنقاذ 🧤
  └─ أُحبط + حركات متبقية → تعزيز بلاعب جديد

---------------------------------------------------------------
قسم 8: نقاط يمكن تطويرها
---------------------------------------------------------------

1. السحب دائماً من أول الكارت:
   المشكلة: لا عشوائية في اختيار الكروت
   التطوير: سحب عشوائي أو بأولوية ذكية

2. الهجوم دائماً بالأعلى هجوماً:
   المشكلة: قابل للتنبؤ من اللاعب
   التطوير: أحياناً يهاجم بلاعب متوسط لخداع اللاعب

3. لا يُفكر في المستقبل:
   المشكلة: ردود فعل آنية فقط، لا تخطيط
   التطوير: إضافة نقاط تقييم استراتيجية للحركات

4. لا يُدير الموارد:
   المشكلة: لا يحتفظ بكروت للطوارئ
   التطوير: إضافة نظام أولويات للموارد

5. الدفاع حتمي بالمزيج الأمثل:
   المشكلة: سهل القراءة والتوقع
   التطوير: إضافة احتمالية خطأ حسب مستوى الصعوبة

6. تأخير ثابت (1200ms):
   المشكلة: يبدو روتينياً ومتوقعاً
   التطوير: تأخير عشوائي بين 800ms-2000ms

7. مستويات الصعوبة لا تؤثر على الذكاء:
   المشكلة: الخوارزمية ثابتة بغض النظر عن normal/tactical/legend
   التطوير: تغيير السلوك حسب مستوى الصعوبة:
     * normal: يرتكب أخطاء بعض الأحيان
     * tactical: الخوارزمية الحالية
     * legend: يُعزز الهجوم مبكراً ولا يُهدر حركات
