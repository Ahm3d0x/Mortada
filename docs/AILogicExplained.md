# 🤖 شرح منطق الكمبيوتر (الذكاء الاصطناعي) بالتفصيل الكامل

> الملف المرجعي: `src/components/GameOffline.tsx`
> الدوال الرئيسية: `handleAIPlayTurn`, `triggerAIDefenseReaction`, `handleConfirmDefense`

---

## 📌 نظرة عامة على الكمبيوتر

الكمبيوتر في اللعبة يُعرَّف بالاسم الثابت:
```
aiCoachName = "المدرب الغريم (تكتيك روبوت)"
aiTeam      = "كتائب الروبوت الذكية 🤖"
```

الكمبيوتر **لا يفكر بشكل مستقل** — بل هو محرك قواعد حتمي يتبع خوارزمية ثابتة في كل دور.
يعمل على مرحلتين رئيسيتين:

| الدور | ما يحدث |
|-------|---------|
| `ai_turn` | الكمبيوتر يلعب دوره الهجومي كاملاً تلقائياً |
| دفاع عند هجوم اللاعب | الكمبيوتر يتحرك تلقائياً للدفاع عند تسديد اللاعب |

---

## 🔁 المرحلة الأولى: دور الكمبيوتر الهجومي (handleAIPlayTurn)

تُطلق هذه الدالة تلقائياً عبر useEffect بمجرد أن تصبح المرحلة ai_turn:

```typescript
useEffect(() => {
  if (phase === "ai_turn" && !isHalfTimeBreak && !isMultiplayer) {
    handleAIPlayTurn();
  }
}, [phase, isHalfTimeBreak, isMultiplayer]);
```

### ⏱️ التسلسل الزمني للدور

```
التشغيل الفوري
    ↓
[تأخير 1200ms] → السحب من المجموعة
    ↓
[تأخير 1200ms إضافية] → تنفيذ الحركات الكاملة
```

---

### 🃏 الخطوة 1: سحب الكروت

الكمبيوتر يسحب ورقتين دائماً:
- ورقة لاعب من aiDeck (إن وُجدت)
- ورقة تكتيك خاصة من specialDeck (إن وُجدت)

ملاحظة: السحب دائماً من أول البطاقة في المجموعة، لا عشوائية هنا.

---

### 🔄 الخطوة 2: المرحلة الأولى — تنظيم التشكيلة

الكمبيوتر يحاول ملء أو تحسين مراكز الملعب بالترتيب التالي:

أ) إذا وجد مركزاً فارغاً
يضع فيه أفضل لاعب في يده (حسب مجموع الهجوم + الدفاع)

ب) إذا لم يجد مركزاً فارغاً لكن وجد لاعباً مستهلكاً (spent)
يستبدل اللاعب المستهلك بأفضل لاعب في اليد ويُعيد اللاعب القديم للمجموعة

ج) إذا لم يجد فارغاً ولا مستهلكاً
يبحث عن أضعف لاعب في الملعب (هجوم < 6 و دفاع < 6)
ثم يتحقق: هل اللاعب الجديد أفضل بنقطتين على الأقل؟
- إذا كان اللاعب الضعيف مكشوفاً → يُطرد نهائياً من الملعب
- إذا كان مقلوباً → يعود ليد الكمبيوتر

المنطق: الكمبيوتر لا يُبدّل إلا إذا كان التغيير يستحق، لتجنب إضاعة الحركات.

---

### 🎴 الخطوة 3: تفعيل الكروت التكتيكية

لكل كارت تكتيكي في يد الكمبيوتر، يُقيّم ما إذا كان يُفيده:

أ) كروت الاستهداف (Destroy/Freeze/Silence/Stun/ReturnToHand/Reveal)
يبحث عن أفضل هدف مكشوف للاعب ويستهدف الأقوى دائماً

ب) كروت السحب (DrawCard / world_cup)
يسحب كروتاً إضافية ويستهلك حركة واحدة

ج) تفعيل قدرات الأسطورة
عند وضع لاعب أسطورة في الملعب، تُفعَّل قدراته تلقائياً:
- AddMoves → زيادة حركات الكمبيوتر
- DrawCard → سحب كروت إضافية

---

### ⚔️ الخطوة 4: المرحلة الثانية — إعلان الهجوم

بعد انتهاء تنظيم التشكيلة، يبحث الكمبيوتر عن أفضل لاعب للهجوم:
- يُرتب المقلوبون غير المستهلكون حسب أعلى هجوم
- يختار الأعلى هجوماً
- شرط: الهجوم غير محظور + يوجد حركات متبقية + قوة هجوم > 1

عند الهجوم:
1. يكشف اللاعب المهاجم (مقلوب → مكشوف)
2. يسحب كارت Booster عشوائي (معزز المرتدة)
3. ينتقل المرحلة إلى ai_attacking
4. يُتاح للإنسان حركات الدفاع

إذا لا يوجد هجوم متاح → ينهي دوره تلقائياً.

---

## 🛡️ المرحلة الثانية: دفاع الكمبيوتر ضد هجوم اللاعب (triggerAIDefenseReaction)

عندما يضغط اللاعب على تسديد/حسم الهجوم، يُستدعى هذا المحرك تلقائياً (تأخير 1200ms).

### 🧮 الخطوة 1: حساب فجوة الدفاع

```
فجوة الدفاع = هجوم اللاعب الكلي - دفاع الكمبيوتر الحالي
```
إذا فجوة الدفاع > 0 → هجوم اللاعب قوي ويحتاج الكمبيوتر للدفاع.

---

### 🎴 الخطوة 2: تقييم الكروت التكتيكية الدفاعية

الكمبيوتر يُرتب الكروت التكتيكية حسب الأولوية الدفاعية:

1. أولوية قصوى: Offside / CancelAction → يلغي الهجوم كلياً!
2. أولوية عالية: FreezeCard/SilenceCard/StunCard/DestroyCard → يضعف أو يحذف المهاجم
3. أولوية متوسطة: park_the_bus / AddStat(defense ≥ 4) → رفع دفاع كبير
4. أولوية عادية: wet_pitch / AddStat(attack ≤ -4) → تخفيض هجوم اللاعب
5. أولوية منخفضة: fans / AddStat(≥ 2) → تعزيز بسيط

إذا كان الكارت من نوع استهداف → يُطبّقه على المهاجم الحالي للاعب تحديداً.

---

### 🔍 الخطوة 3: خوارزمية اختيار أفضل مزيج دفاعي

بعد تفعيل الكروت التكتيكية، يحسب الكمبيوتر فجوة الدفاع المتبقية ويحاول إيجاد أفضل مزيج:

◆ محاولة بلاعب واحد:
يبحث عن أقل لاعب كافٍ (دفاع >= فجوة الهجوم) لا يُبدد حركات زائدة

◆ إذا فشل → محاولة بلاعبين:
يجرب كل الأزواج الممكنة ويختار الزوج بأقل فائض

◆ إذا فشل → محاولة بثلاثة لاعبين:
نفس المنطق مع 3 لاعبين

◆ إذا فشل → محاولة بأربعة لاعبين:
يجرب كل التوليفات الممكنة من 4 لاعبين

◆ إذا فشل الكل → يكشف الأقوى دفاعاً (Fallback):
يفرز حسب الدفاع ويكشف أعلى عدد مسموح به

◆ إذا لا يوجد مدافعون:
"الخصم أدرك استحالة الصد وفضّل الحفاظ على أوراقه لهجمة مضادة لاحقاً!"

القيد المهم: عدد اللاعبين المكشوفين = maxMovesPerTurn - عدد الكروت التكتيكية المُفعَّلة

---

## 🔢 المرحلة الثالثة: لحظة الحسم (handleConfirmDefense)

عندما يضغط اللاعب على تأكيد الدفاع:

### 🎯 تعزيز هجومي من الكمبيوتر (لحظة الحسم)

قبل إعلان النتيجة، إذا كان الكمبيوتر سيخسر:
يضيف الكمبيوتر كروتاً هجومية من يده واحدة بواحدة ويُعيد الحساب بعد كل كارت.

### 📊 معادلة الحسم النهائية

```
هجوم الكمبيوتر = قيمة المهاجم + قيمة المعزز (Booster) + تأثيرات الكروت التكتيكية + لاعبون داعمون + قدرات خاصة
دفاع اللاعب   = مجموع قيم المدافعين المكشوفين + تأثيرات الكروت التكتيكية الدفاعية + قدرات خاصة

إذا هجوم الكمبيوتر > دفاع اللاعب → هدف للكمبيوتر 🥅
إذا هجوم الكمبيوتر ≤ دفاع اللاعب → إنقاذ للاعب 🧤
```

### 🔄 تعزيز الهجوم المضاعف (AI Reinforce Attack)

إذا أُحبط هجوم الكمبيوتر لكن لا يزال لديه حركات:
يكشف أعلى لاعب هجوماً من المقلوبين غير المستهلكين
ثم تعود المرحلة إلى ai_attacking ليدافع اللاعب مجدداً!

## 🏗️ محرك حساب القوة (rulesEngine.ts)

### قوة الهجوم تشمل:
| المصدر | الوصف |
|--------|--------|
| card.attack | قيمة اللاعب الأساسية |
| currentBooster.value | قيمة كارت المعزز العشوائي |
| AddStat(attack) | تعزيز من الكروت التكتيكية |
| لاعبون داعمون مكشوفون | كل لاعب داعم يُضيف قوته |
| قدرات خاصة مفعّلة | حسب نوعها |

### قوة الدفاع تشمل:
| المصدر | الوصف |
|--------|--------|
| مجموع card.defense للمدافعين المكشوفين | القاعدة الأساسية |
| AddStat(defense) | تعزيز من الكروت التكتيكية |
| park_the_bus / BlockAttack | تأثيرات حظر |
| قدرات خاصة للأسطورة | حسب نوعها |

---

## 🔒 القيود والضمانات

| القيد | الوصف |
|-------|--------|
| isAIExecutingRef | يمنع تشغيل دورين للكمبيوتر في نفس الوقت |
| isHalfTimeBreakRef | يوقف الكمبيوتر فوراً عند استراحة الشوط |
| aiTimeoutId1 / aiTimeoutId2 | مراجع للـ timeouts يمكن إلغاؤها |
| maxMovesPerTurn | سقف الحركات في كل دور |
| isAttackBlockedFor(false) | يتحقق إذا كان الهجوم محظوراً |
| isSpecialCardsBlocked(false) | يتحقق إذا كانت الكروت التكتيكية محظورة |

## 📈 ملخص الخوارزمية الكاملة

```
بداية دور الكمبيوتر (ai_turn)
         │
[1200ms تأخير]
         │
◆ سحب ورقتين (لاعب + تكتيك)
         │
[1200ms تأخير]
         │
◆ مرحلة 1: تنظيم التشكيلة
  - مركز فارغ؟ → ضع أفضل لاعب من اليد
  - مركز مستهلك؟ → استبدله بأفضل لاعب
  - لاعب ضعيف (attack<6)؟ → استبدله إذا الجديد أفضل بـ 2 نقاط+
         │
◆ تفعيل الكروت التكتيكية
  - كروت استهداف → أقوى لاعب مكشوف للاعب
  - كروت سحب → سحب إضافي
         │
◆ مرحلة 2: إعلان الهجوم
  - يوجد مقلوب بهجوم > 1؟ → يختار الأعلى هجوماً
  - يكشفه + يسحب Booster → ينتقل إلى ai_attacking
  - لا؟ → ينهي الدور
         │
    [عند هجوم الكمبيوتر - triggerAIDefenseReaction]
         │
◆ يحسب فجوة الدفاع
◆ يُفعّل كروت تكتيكية دفاعية (بالأولوية)
◆ يجرب مزيج 1→2→3→4 لاعبين
◆ يكشف الأفضل إذا فشل كل المحاولات
         │
    [عند تأكيد الدفاع من اللاعب - handleConfirmDefense]
         │
◆ كمبيوتر يُضيف كروت هجومية (إذا كان سيخسر)
◆ الحساب: هجوم vs دفاع
◆ هجوم > دفاع → هدف 🥅
◆ هجوم ≤ دفاع → إنقاذ 🧤
◆ إذا أُحبط + حركات متبقية → تعزيز بلاعب جديد
```

## 💡 نقاط يمكن تطويرها

| نقطة الضعف | السبب | اقتراح التطوير |
|------------|-------|----------------|
| السحب دائماً من أول الكارت | لا عشوائية في الاختيار | سحب عشوائي أو بأولوية |
| الهجوم دائماً بالأعلى هجوماً | قابل للتنبؤ | أحياناً يهاجم بلاعب متوسط مفاجأةً |
| لا يُفكر في المستقبل | ردود فعل آنية فقط | إضافة نقاط تقييم استراتيجية |
| لا يُدير الموارد | لا يحتفظ بكروت للطوارئ | إضافة نظام أولويات |
| الدفاع حتمي بالمزيج الأمثل | سهل القراءة | إضافة احتمالية خطأ حسب الصعوبة |
| تأخير ثابت (1200ms) | يبدو روتينياً | تأخير عشوائي بين 800ms-2000ms |
| مستويات الصعوبة لا تؤثر على الذكاء | الذكاء ثابت | تغيير السلوك حسب normal/tactical/legend |
